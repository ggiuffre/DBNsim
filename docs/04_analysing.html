Analysing the DBN architecture

You can analyse a DBN in several ways. The simplest and the first thig to do when you set up a DBN for learning a dataset is analysing its architecture. The architecture of a network is the way in which its neurons are connected; in a DBN, the architecture is defined by:

  • the number of hidden layers, that is equal to the number of RBMs forming the DBN;
  • the number of units in each layer.

For a straightforward representation of the architecture, you can look at the graph in the upper-right corner.

Plotting the recontruction error

In the bottom-left corner, you can see a chart reporting the reconstruction error for each RBM and for each training epoch.
